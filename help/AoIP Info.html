Introduction

The AoIP plugin shows information about any connected Audio Over IP session rather than the actual audio. Note if the selected input is Soundcard or File rather than RTP then this plugin will show some information about the selected input.

The plugin does not go full screen so the Screen Selection and Option Selection buttons are always visible.

The plugin has 4 pages of information selectable by clicking on the Tabs at the top of the screen.

Session

Shows the overall Session details.

  • Input - the name of the connection as defined/discovered in the main AoIP Settings page. This will be empty if the input is not set to RTP
  • Name -
    • If the input is RTP this will show the name of the sesson as defined in the SDP
    • If the input is Soundcard this will show the selected soundcard input
    • If the input is File this will show the filename
  • Type - the type of connection.
    • If the input is RTP this will show either Mulitcast or Unicast.
    • If the input is Soundcard this will say Soundcard
    • If the input is File this will say File
  • Description - any desciption for the session as defined in the SDP
  • Sync Type/Version/ID/Domain - Synchronization details
  • Audio Subsessions - the number of audio subsessions that this Session contains
  • Video Subsessions - the number of vide subsession this Session contains
  • Duplicate Streams - SMPTE 2110 allows duplicate audio and video streams in a single session.

Active Subsession

Shows information about the currently connected Subsession

  • Source
    • If the input is RTP this will show the IP address of the incoming audio. This may be a Unicast or Multicast address
    • If the input is Soundcard this will show the selected soundcard input.
    • If the input is File this will show the filename
  • Codec - this will show the RTP audio codec. PAM can currently decode L24 (PCM 24 bit) and L16 (PCM 16 bit).
  • Frequency - this is the sample rate of the incoming audio
  • Channels - this is the number of channels in the incoming audio. PAM can currently cope with 0-8 channels.
  • Id - this is the RTP id of the active subsession
  • Sync Type/Version/Sync ID/Domain - Synchronization details.
  • Sync Timestamp - all RTP frames have a 32 bit count in the header. The sync timestamp is the timestamp that the RTP frame count would be 0.
  • Current Timestamp - this is the timestamp of the incoming RTP header.
  • Transmission Time - this is the time that the incoming RTP header was transmitted from the sender.
  • Presentation Time - this is the time that the incoming RTP header was received by PAM
  • Network Latency - this is the difference between the Presentation Time and the Transmission Time.
  • Display/Playback Time - this is the time that the audio was displayed on the screen and (if enabled) played out.
  • Playback latency - this is the difference between the Display/Playback Time and the Presentation Time.
  • Last Frame Size - this is the number of bytes received in the last RTP frame
  • Last Frame Duration - this is the amount of audio (in time) received in the last RTP frame.

QoS

Shows information about the quality of the incoming RTP stream. The QoS measurements are taken at preset intervals. The user can change the interval by selecting the Settings option page (see below).

  • QoS Measurement Time - shows how long the QoS measurements have been running
  • Total Packets Received - shows the total number of RTP packets received.
  • Total Packets Lost - shows the total number of RTP packets lost.
  • kBits/s Min - shows the minimum bitrate of the incoming RTP stream.
  • kBits/s Average - shows the average bitrate of the incoming RTP stream.
  • kBits/s Max - shows the maximum bitrate of the incoming RTP stream.
  • Packets Lost/s Min - show the mnimum number of packets lost per second.
  • Packets Lost/s Average - shows the average nunmber of packets lost per second.
  • Packet Lost/s Max - show the maximum number of packets lost per second.
  • Inter Packet Gap Min - shows the minumum amount of time between two adjacent packets being received.
  • Inter Packet Gap Average - shows the average amount of time between two adjacent packets being received.
  • Inter Packet Gap Max - shows the maximum amount of time between two adjacent packets being received.
  • Jitter - shows the current variation in packet delay in ms.
  • Current Graph - shows what the graph below is currently showing

Below the labels is a graph showing a history of the selected QoS measurement.

  • The graph moves from right to left. So the right of the graph is current time and the further to the left the further back in time.
  • The labels on the left of the graph show the maximum and minimum values of the selected QoS measurement.
  • The coloured line on the graph shows the moment by moment value and the white line (if shown) shows the average value.

Raw SDP


Shows the raw SDP description for the session as received by PAM.
